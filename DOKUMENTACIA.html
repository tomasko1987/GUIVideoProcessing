Metóda: Log(LogLevel level, string message)

public void Log(LogLevel level, string message)
{
    if (level < _minLevel) return;

    string ts = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss.fff");
    string line = $"{ts} | {level.ToString().ToUpper()} | {message}";

    if (!_queue.TryAdd(line))
    {
        // Fronta plná - správa sa zahodí (fail-safe)
    }
}

Charakteristika:

  • Neblokujúca operácia (TryAdd namiesto Add)
  • Ak je fronta plná → správa sa zahodí (nepochytateľné)
  • Formát timestamp: yyyy-MM-dd HH:mm:ss.fff
  • Formát riadku: timestamp | LEVEL | message

Background task: WriterLoop()

private void WriterLoop(CancellationToken token)
{
    try
    {
        foreach (var line in _queue.GetConsumingEnumerable(token))
        {
            RotateIfNeeded();      // Kontrola rotácie
            _writer.WriteLine(line); // Zápis
            _writer.Flush();         // Okamžitý flush
        }
    }
    catch (OperationCanceledException) { }
    catch { }
    finally
    {
        _writer?.Dispose();
    }
}

Vlastnosti:

  • Beží na background threade (Task.Run)
  • Číta z BlockingCollection (blokuje keď je prázdna)
  • Kontroluje rotáciu pred každým zápisom
  • Flush po každom riadku (bezpečnosť pri crashe)
  • Korektné ukončenie cez CancellationToken

Rotácia log súborov

Dva typy rotácie:

  1. Denná rotácia: Každý deň nový súbor log_YYYY-MM-DD.txt
  2. Rotácia podľa veľkosti: Ak súbor presiahne 5 MB → log_YYYY-MM-DD_001.txt

Metóda: RotateIfNeeded()

private void RotateIfNeeded()
{
    lock (_rotateLock)
    {
        // 1) Denná rotácia
        DateTime today = DateTime.Today;
        if (today != _currentDate)
        {
            _currentDate = today;
            _writer?.Dispose();
            OpenWriterForDate(_currentDate);
            return;
        }

        // 2) Rotácia podľa veľkosti
        var info = new FileInfo(_currentFilePath);
        if (info.Exists && info.Length >= _maxBytesPerFile)
        {
            _writer?.Dispose();
            OpenWriterForDate(_currentDate);
        }
    }
}

ProcessFrame(Mat frame) : Mat

Účel: Extrakcia ROI z frame, kreslenie obdĺžnika

Algoritmus:

  1. Načítanie ROI parametrov z UI (percentá) 
    double px = (double)nudX.Value / 100.0;
double py = (double)nudY.Value / 100.0;
double pw = (double)nudWidth.Value / 100.0;
double ph = (double)nudHeight.Value / 100.0;
  2. Prepočet na pixely 
    int x = (int)(frame.Width * px);
int y = (int)(frame.Height * py);
int rw = (int)(frame.Width * pw);
int rh = (int)(frame.Height * ph);
  3. Validácia (ochrana proti out-of-bounds)
    • x, y >= 0
    • x + rw <= frame.Width
    • y + rh <= frame.Height
    • rw, rh > 0
  4. Extrakcia ROI 
    var roiRect = new Rect(x, y, rw, rh);
lock (_roiLock)
{
    _lastRoi?.Dispose();
    _lastRoi = new Mat(frame, roiRect).Clone();
}
  5. Kreslenie obdĺžnika 
    Mat display = frame.Clone();
Cv2.Rectangle(display, roiRect, new Scalar(0, 255, 0), 2);
return display;

Up() : void

Účel: Zväčšenie ROI pre detailnejšie spracovanie

private void Up()
{
    Mat roiCopy = null;
    double scale = (double)GetNudValueSafe(nudResize);

    lock (_roiLock)
    {
        if (_lastRoi == null || _lastRoi.Empty()) return;
        roiCopy = _lastRoi.Clone();
    }

    try
    {
        Mat upTemp = new Mat();
        Cv2.Resize(roiCopy, upTemp, new Size(), scale, scale,
                   InterpolationFlags.Cubic);

        lock (_roiLock)
        {
            _lastUp?.Dispose();
            _lastUp = upTemp;
            upTemp = null;
        }
    }
    finally
    {
        roiCopy?.Dispose();
        upTemp?.Dispose();
    }
}

Vlastnosti:

  • Thread-safe (Clone + locks)
  • Interpolácia: Cubic (vysoká kvalita pre OCR)
  • Scale faktor: 1×, 3×, 5× (nastaviteľné)

Pipeline() : void

Účel: Image processing pipeline pre prípravu OCR

Kroky algoritmu:

_lastUp (zväčšený ROI) │ ├─► 1. Grayscale (BGR → GRAY) │ Cv2.CvtColor(BGR2GRAY) │ ├─► 2. Bilateral Filter (noise reduction + edge preservation) │ Cv2.BilateralFilter(d:9, sigmaColor:50, sigmaSpace:50) │ ├─► 3. CLAHE (Contrast Limited Adaptive Histogram Equalization) │ clipLimit: 2.0, tileGridSize: 8×8 │ → _lastContrast (debug) │ ├─► 4. Gaussian Blur (zjemnenie) │ Cv2.GaussianBlur(3×3) │ ├─► 5. Adaptive Threshold (BinaryInv) │ blockSize: 41 (nepárne), C: 5 │ → pozadie biele, čísla čierne │ → _lastBinary (debug) │ ├─► 6. Morphology Open (odstránenie bodiek) │ kernel: 3×3 Rect, iterations: 1 │ ├─► 7. Morphology Close (vyplnenie dier) │ kernel: 3×3 Rect, iterations: 1 │ └─► 8. Connected Components + Area filtering minArea: 350 (nastaviteľné) → odstránenie malých komponentov (šum) → _lastCleaned (finálny výsledok)

Prečo tento pipeline?

  • Bilateral Filter: Zníži šum, ale zachová hrany čísel
  • CLAHE: Lokálne zvýši kontrast (lepšie než globálny threshold)
  • Adaptive Threshold: Funguje aj pri nerovnomernom osvetlení
  • Morphology: Odstráni drobné artefakty
  • Area filtering: Odstráni bodky a šum (len veľké objekty = čísla)

Thread-safety mechanizmy

Kritické sekcie

Lock Chráni Problém bez locku
_capLock _cap.Read() vs _cap.Dispose() AccessViolationException (crash)
_roiLock _lastRoi, _lastUp, _lastContrast, atď. _dims error, memory corruption

Príklad problému bez _capLock:

// CAPTURE THREAD              // UI THREAD (STOP)
_cap.Read(frame) ────────┐
  ├─ native call        │   StopCapture()
  │  (FFmpeg decode)    │     _cap.Dispose() ◄── CRASH!
  └─ AccessViolation ◄──┘       (native free)

// RIEŠENIE: lock pred Read + Dispose
lock (_capLock) {
    if (_cap == null) break;
    ok = _cap.Read(frame);
}

Metóda Load

public static Settings Load(string filePath)

Účel: Načíta nastavenia zo súboru Settings.json

Algoritmus:

  1. Kontrola existencie súboru 
    if (!File.Exists(filePath))
  2. Ak neexistuje:
    • Vytvorí nový Settings s prázdnym listom
    • Zavolá Save() → vytvorí súbor na disku
    • Vráti nový objekt
  3. Ak existuje:
    • Prečíta obsah: File.ReadAllText(filePath)
    • Deserializuje: JsonSerializer.Deserialize<Settings>(json)
    • Validuje (null checks)
    • Vráti načítaný objekt

Ochranné mechanizmy

// Ochrana 1: Deserializácia zlyhala
if (settings == null)
{
    settings = new Settings();
}

// Ochrana 2: IPAddress property je null

{
    settings.IPAddress = new List<string>();
}

Tieto ochrany zabezpečujú, že aplikácia nikdy nespadne na NullReferenceException, aj keď je JSON súbor poškodený.

Zmena 3: Rozšírenie Pipeline() - krok 9 (riadok 1145-1183)

PRIDANÉ do Pipeline() po kroku 8
// ========================= 9) NOVÝ KROK: ChangeMatrix - Resize cleaned na NxN =========================
// Tento krok vezme cleaned Mat a zmenší ho na fixnú veľkosť NxN (16x16, 32x32, 64x64 alebo 128x128).
// Účel: Príprava vstupu pre neurónové siete alebo pattern matching (normalizovaná veľkosť).

// Prečítaj vybranú veľkosť matice z ComboBoxu (thread-safe).
// GetComboValueSafe() volá Invoke() pre bezpečný prístup z background threadu.
int matrixSize = GetComboValueSafe(cboChangeMatrix);

// Ochrana: ak je matrixSize <= 0 (chyba alebo nič nevybrané), preskočíme resize.
// Toto by sa nemalo stať, lebo v konštruktore nastavujeme default hodnotu 32.
if (matrixSize > 0)
{
    // Vytvoríme dočasný Mat pre resized výsledok.
    Mat matrix = new Mat();

    // Resize cleaned Mat na NxN pixelov.
    // - cleaned: vstupný Mat (čierne pozadie, biele objekty)
    // - matrix: výstupný Mat (resized)
    // - new Size(matrixSize, matrixSize): cieľová veľkosť (napr. 32x32)
    // - InterpolationFlags.Area: najlepšia interpolácia pre zmenšovanie (zachováva detaily)
    Cv2.Resize(cleaned, matrix, new OpenCvSharp.Size(matrixSize, matrixSize), 0, 0, InterpolationFlags.Area);

    // Uloženie resized matice do _lastMatrix (thread-safe pod _roiLock).
    lock (_roiLock)
    {
        // Uvoľni starý _lastMatrix (ak existuje) aby nevznikol memory leak.
        _lastMatrix?.Dispose();

        // Ulož nový resized Mat do _lastMatrix.
        // Matrix už má vlastnú pamäť (nie je SubMat), takže nepotrebujeme Clone().
        _lastMatrix = matrix;

        // Nastavíme matrix na null, aby ho finally blok náhodou neDispose-nul.
        // (matrix teraz vlastní _lastMatrix)
        matrix = null;
    }

    // Poznámka: matrix?.Dispose() v finally bloku uvoľní matrix len ak nebol priradený do _lastMatrix.
}

Podrobné vysvetlenie algoritmu:

1. Čítanie veľkosti matice (thread-safe)

int matrixSize = GetComboValueSafe(cboChangeMatrix);
  • Pipeline() beží na background threade (CaptureLoop)
  • cboChangeMatrix je UI kontrolka (žije na UI threade)
  • Priamy prístup by vyvolal CrossThreadException
  • GetComboValueSafe() použije Invoke() pre bezpečný prístup
  • Vráti int (16, 32, 64 alebo 128)

2. Validácia veľkosti

if (matrixSize > 0)
  • Ochrana proti neplatným hodnotám (0 alebo záporné)
  • V praxi by to nemalo nastať (default je 32)
  • Defenzívne programovanie - fail-safe

3. Vytvorenie výstupného Matu

Mat matrix = new Mat();
  • Prázdny Mat pre výsledok resize operácie
  • Cv2.Resize() ho automaticky alokuje

4. Resize operácia

           0, 0, InterpolationFlags.Area);

cleaned Vstupný Mat Finálny výsledok z kroku 8 (čierne pozadie, biele objekty)
matrix Výstupný Mat Sem sa uloží resized výsledok

Interpolácia Použitie Kvalita
Nearest Najrýchlejšia, pixelizovaná ❌ Zlá pre zmenšovanie
Linear Bilineárna, rýchla ⚠️ Stredná

{
    _lastMatrix?.Dispose();

/// Bezpečne prečíta vybranú hodnotu z ComboBoxu aj vtedy, keď je volaná z iného vlákna než UI.
/// Používa sa na čítanie cboChangeMatrix z background threadu (CaptureLoop -> Pipeline).
/// </summary>

private int GetComboValueSafe(ComboBox cbo)
{
    // Ak sme na UI threade (InvokeRequired == false), môžeme hodnotu čítať priamo.
    if (!cbo.InvokeRequired)
    {
        // Priamy návrat vybranej hodnoty z ComboBoxu.
        // SelectedItem je object, musíme ho skonvertovať na int.
        // Ak nie je nič vybrané (SelectedItem == null), vrátime 0.
        return cbo.SelectedItem != null ? (int)cbo.SelectedItem : 0;
    }

    // Ak sme na inom (ne-UI) vlákne, musíme si hodnotu vypýtať cez Invoke na UI thread.
    // Invoke je blokujúci – počká, kým UI thread hodnotu prečíta a vráti.
    return (int)cbo.Invoke(new Func<int>(() =>
    {
        // Tento kód už beží na UI threade, takže čítanie SelectedItem je bezpečné.
        // SelectedItem je object (môže byť int 16, 32, 64, 128), skonvertujeme na int.
        // Ochrana: ak by bolo null, vrátime 0.
        return cbo.SelectedItem != null ? (int)cbo.SelectedItem : 0;
    }));
}

Zmena 5: CaptureLoop() - zobrazenie picMatrix (riadok 689-746)

PRIDANÉ po sekcii picCleaned
// ========================= NOVÉ: Zober kópiu matrix a sprav z toho Bitmap =========================
// Rovnaký pattern ako pre cleaned – thread-safe prístup cez Clone.
Bitmap bmpMatrix = null;

// Zoberieme snapshot _lastMatrix pod lockom (clone), aby sme nikdy nekonvertovali zdieľaný Mat.
Mat matrixCopy = null;

lock (_roiLock)
{
    // Ak _lastMatrix existuje a nie je prázdny, spravíme kópiu.
    if (_lastMatrix != null && !_lastMatrix.Empty())
    {
        matrixCopy = _lastMatrix.Clone();
    }
}

// Ak máme matrixCopy, prevedieme ho na Bitmap mimo locku.
if (matrixCopy != null)
{
    try
    {
        // BitmapConverter.ToBitmap() konvertuje Mat → Bitmap (GDI+).
        // Matrix je malý (napr. 32x32), ale PictureBox ho zväčší cez Zoom mode.
        bmpMatrix = BitmapConverter.ToBitmap(matrixCopy);
    }
    finally
    {
        // Uvoľníme lokálnu kópiu matrixCopy (už ju nepotrebujeme).
        matrixCopy.Dispose();
    }
}

Pattern je identický ako pre cleaned:

  1. Clone pod lockom: Vytvorí bezpečnú kópiu _lastMatrix
  2. Konverzia mimo locku: BitmapConverter.ToBitmap() beží mimo kritickej sekcie
  3. Dispose v finally: Uvoľní lokálnu kópiu (prevencia leak)
PRIDANÉ do UI Update (Invoke blok)
// ====== picMatrix (NOVÉ) ======
// Ak sme dostali matrix bitmapu (resized 16x16, 32x32, 64x64, 128x128), zobrazíme ju.
// PictureBox má SizeMode = Zoom, takže malá matica sa zväčší na celý PictureBox.

    picMatrix.Image?.Dispose();
    // Nastavíme nový obrázok (bmpMatrix vlastníme my, NEclone).

Vysvetlenie UI update:

  • Celý kód beží v Invoke() bloku → bezpečné pre UI thread
  • picMatrix.Image?.Dispose(): Uvoľní starý obrázok (prevencia leak)
  • picMatrix.Image = bmpMatrix: Nastaví nový obrázok
  • PictureBox má SizeMode = Zoom → malý obrázok (napr. 32×32) sa zväčší proporcionálne

Prečo NEclone pri priradení?

Na rozdiel od picFrame.Image = (Bitmap)bmpFrame.Clone(), tu NEclonujeme:

  • bmpMatrix je lokálna premenná vytvorená v tomto threade
  • Nie je zdieľaný medzi threadmi
  • Po priradení do picMatrix.Image vlastníctvo prechádza na PictureBox
  • PictureBox ho automaticky Dispose-ne pri ďalšom update alebo pri zatvorení formulára

Časť A: Dispose _lastMatrix (riadok 549-552)

Objekt Typ pamäte Dôsledok ak sa neuvoľní

4.1.1 Identifikované Problémy v Pôvodnom Pipeline (Kapitola 3)

❌ PROBLÉM #1: Duplicitné Smoothing

Bilateral Filter (krok 2) a Gaussian Blur (krok 4) vykonávajú podobnú funkciu. Bilateral Filter už robí high-quality smoothing s edge preservation, takže Gaussian Blur je zbytočný a stráca detaily číslic.

  • blockSize=41 v Adaptive Threshold je príliš veľký pre malé číslice → optimálne 15
  • CLAHE tileGridSize=(8,8) príliš veľké pre malé číslice → lepšie (4,4)
  • Bilateral sigma=50 môže byť silnejší → 75
  • CLAHE clipLimit=2.0 slabý kontrast → 3.0
  • minArea=350 príliš veľké pre malé číslice → 200

Všetky pipeline parametre boli hardcoded v kóde. Používateľ nemohol meniť parametre bez rekompilácie aplikácie.
