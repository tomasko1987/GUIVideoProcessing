📊 Optimalizácia Image Processing Pipeline pre Rozpoznávanie Dvojice Číslic

📌 Cieľ analýzy: Optimalizovať existujúci Image Processing pipeline v metóde Pipeline() pre čo najlepšie rozpoznávanie dvojice digitálnych číslic (napr. "42", "15", "99") z ROI oblasti ESP32-CAM streamu.

1. Analýza Aktuálneho Pipeline

1.1 Prehľad Krokov

Aktuálny Image Processing pipeline obsahuje 9 krokov:

Krok Operácia Parametre Účel
1 Gray Conversion BGR2GRAY Konverzia na grayscale (1 kanál)
2 Bilateral Filter d=9, sigmaColor=50, sigmaSpace=50 Odstránenie šumu, zachovanie hrán
3 CLAHE clipLimit=2.0, tileGridSize=(8,8) Lokálne zvýšenie kontrastu
4 Gaussian Blur kernelSize=(3,3), sigma=0 Zjemnenie pred thresholdom
5 Adaptive Threshold GaussianC, BinaryInv, blockSize=41, C=5.0 Binarizácia (pozadie biele, číslice čierne)
6 Morphology Open + Close, kernel 3x3, iterations=1 Odstránenie bodiek, vyplnenie dier
7 Connected Components minArea=350 Filtrovanie malých objektov (smeti)
8 ChangeMatrix Resize na NxN (16/32/64/128) Normalizácia veľkosti pre OCR/NN

1.2 Identifikované Problémy

❌ KRITICKÝ PROBLÉM #1: Duplicitné Smoothing

Bilateral Filter (krok 2) a Gaussian Blur (krok 4) vykonávajú podobnú funkciu (smoothing/rozmazanie). Bilateral už robí vysoký-kvalitný smoothing s zachovaním hrán, takže Gaussian Blur je ZBYTOČNÝ a znižuje kvalitu (stráca detaily číslic).

⚠️ PROBLÉM #2: Neoptimálne Parametre pre Číslice
  • blockSize=41 v Adaptive Threshold je príliš VEĽKÝ pre malé číslice. Pre rozpoznávanie číslic v ROI optimálna hodnota je 11-21.
  • CLAHE tileGridSize=(8,8) je príliš veľké pre malé číslice. Lepšie (4,4) alebo (2,2).
  • Bilateral sigmaColor/sigmaSpace=50 môže byť zvýšené na 75 pre silnejšie odstránenie šumu pri zachovaní hrán.
⚠️ PROBLÉM #3: Chýbajúce Kroky pre Segmentáciu

Pipeline nemá krok na segmentáciu jednotlivých číslic. Pre rozpoznávanie dvojice číslic (napr. "42") je potrebné:

  • Nájsť kontúry jednotlivých číslic
  • Vytvoriť bounding boxy (obdĺžniky okolo každej číslice)
  • Extrahovať jednotlivé číslice ako samostatné Mat objekty
  • Zoradiť číslice zľava doprava
ℹ️ PROBLÉM #4: Nedostatok Vizualizácie Medzikrokov

Pre ladenie a optimalizáciu parametrov by bolo užitočné vidieť výsledky jednotlivých krokov v UI (napr. pridať PictureBox pre kontúry, edge detection, atď.).

2.2 Optimalizácia Variant B: Pokročilá (s Kontúrami)

📌 POKROČILÁ STRATÉGIA

Táto varianta pridáva krok 8A: Segmentácia Číslic medzi Connected Components a ChangeMatrix. Umožní extrahovať jednotlivé číslice, zoradiť ich a pripraviť pre OCR/NN rozpoznávanie.

Nový Krok 8A: Segmentácia Číslic (Contours + BoundingBox)

// ========================= 8A) NOVÝ KROK: Segmentácia Číslic =========================
// Tento krok nájde kontúry jednotlivých číslic a extrahuje ich ako samostatné Mat objekty.

// Nájdeme všetky kontúry v cleaned obraze.
Point[][] contours;
HierarchyIndex[] hierarchy;
Cv2.FindContours(cleaned, out contours, out hierarchy,
    RetrievalModes.External, ContourApproximationModes.ApproxSimple);

// List pre uloženie nájdených číslic (zoradené zľava doprava).
List<DigitInfo> digits = new List<DigitInfo>();

// Pre každú kontúru vytvoríme bounding box.
foreach (var contour in contours)
{
    // Vypočítame bounding rectangle okolo kontúry.
    Rect boundingBox = Cv2.BoundingRect(contour);

    // Filter: ignorujeme príliš malé alebo príliš veľké objekty.
    int area = boundingBox.Width * boundingBox.Height;
    if (area < 200 || area > 10000)
        continue;

    // Filter: číslice majú typicky aspect ratio 1.5-3.5 (vyššie než širšie).
    double aspectRatio = (double)boundingBox.Height / boundingBox.Width;
    if (aspectRatio < 1.0 || aspectRatio > 4.0)
        continue;

    // Extrahovanie ROI pre konkrétnu číslicu.
    Mat digitRoi = new Mat(cleaned, boundingBox).Clone();

    // Uložíme informácie o číslici (pozícia X pre zoradenie).
    digits.Add(new DigitInfo
    {
        BoundingBox = boundingBox,
        DigitMat = digitRoi,
        X = boundingBox.X
    });
}

// Zoradíme číslice zľava doprava (podľa X súradnice).
digits = digits.OrderBy(d => d.X).ToList();

// Teraz máme List<DigitInfo> s jednotlivými číslicami zoradeními zľava doprava.
// Môžeme ich poslať do OCR alebo neurónové siete.

// Helper class pre uloženie informácií o číslici.
public class DigitInfo
{
    public Rect BoundingBox { get; set; }  // Pozícia v obraze
    public Mat DigitMat { get; set; }      // Extrahovaná číslica
    public int X { get; set; }             // X súradnica (pre zoradenie)
}

Vizualizácia Kontúr v UI

6. Kompletný Upravený Kód - Variant A

6.1 Pipeline() Metóda (MainForm.cs)

Kompletný kód Pipeline() s aplikovanými zmenami Variant A:

private void Pipeline()
{
    // Lokálna kópia "up" (snapshot), aby sa počas spracovania nezmenil/neudisposoval.
    Mat upCopy = null;

    // Zober snapshot _lastUp pod lockom.
    lock (_roiLock)
    {
        // Ak up ešte nie je k dispozícii, pipeline nemá čo robiť.
        if (_lastUp == null)
            return;

        // Ak je up prázdny, skončíme (inak by padli OpenCV operácie).
        if (_lastUp.Empty())
            return;

        // Clone = vlastná pamäť pre bezpečné spracovanie mimo locku.
        upCopy = _lastUp.Clone();
    }

    // Ak sa nepodarilo spraviť kópiu, skončíme.
    if (upCopy == null || upCopy.Empty())
    {
        upCopy?.Dispose();
        return;
    }

    // Dočasné Mat-y pre jednotlivé kroky pipeline.
    Mat gray = null;
    Mat smooth = null;
    Mat contrast = null;
    // smoothGaussian ODSTRÁNENÉ (duplicitné s Bilateral Filter)
    Mat binary = null;
    Mat cleaned = null;

    try
    {
        // 1) Gray – prevedenie na 1-kanálový obraz.
        gray = new Mat();
        Cv2.CvtColor(upCopy, gray, ColorConversionCodes.BGR2GRAY);

        // 2) Bilateral filter – zníží šum, ale nechá hrany číslic.
        // ZMENENÉ: sigmaColor 50→75, sigmaSpace 50→75 (silnejšie odstránenie šumu)
        smooth = new Mat();
        Cv2.BilateralFilter(gray, smooth, d: 9, sigmaColor: 75, sigmaSpace: 75);

        // 3) CLAHE – lokálne zvýšenie kontrastu.
        // ZMENENÉ: clipLimit 2.0→3.0 (silnejší kontrast), tileGridSize (8,8)→(4,4) (jemnejšia lokálnosť)
        var clahe = Cv2.CreateCLAHE(clipLimit: 3.0, tileGridSize: new OpenCvSharp.Size(4, 4));
        contrast = new Mat();
        clahe.Apply(smooth, contrast);

        // 4) Gaussian blur – ODSTRÁNENÉ (duplicitné s Bilateral Filter)
        // smoothGaussian = new Mat();
        // Cv2.GaussianBlur(contrast, smoothGaussian, new OpenCvSharp.Size(3, 3), 0);

        // 5) Adaptive threshold – pozadie biele, čísla čierne (BinaryInv).
        // ZMENENÉ: vstup smoothGaussian→contrast (priame použitie CLAHE výstupu)
        int blockSize = (int)GetNudValueSafe(nudBlockSize);

        // blockSize MUSÍ byť nepárne a >= 3
        if (blockSize < 3) blockSize = 3;
        if (blockSize % 2 == 0) blockSize += 1;

        double c = (double)GetNudValueSafe(nudC);

        binary = new Mat();
        Cv2.AdaptiveThreshold(
            contrast,  // ZMENENÉ: bol smoothGaussian
            binary,
            maxValue: 255,
            adaptiveMethod: AdaptiveThresholdTypes.GaussianC,
            thresholdType: ThresholdTypes.BinaryInv,
            blockSize: blockSize,
            c: c
        );

        // 6) Morphology – odstráni bodky + vyplní malé diery.
        Mat kernel = Cv2.GetStructuringElement(MorphShapes.Rect, new OpenCvSharp.Size(3, 3));
        Cv2.MorphologyEx(binary, binary, MorphTypes.Open, kernel, iterations: 1);
        Cv2.MorphologyEx(binary, binary, MorphTypes.Close, kernel, iterations: 1);

        // 7) Connected components – necháme len veľké objekty (číslice), malé smeti zahodíme.
        Mat labels = new Mat();
        Mat stats = new Mat();
        Mat centroids = new Mat();

        try
        {
            int numLabels = Cv2.ConnectedComponentsWithStats(
                binary, labels, stats, centroids, PixelConnectivity.Connectivity8);

            cleaned = Mat.Zeros(binary.Size(), MatType.CV_8UC1);
            int statArea = (int)ConnectedComponentsTypes.Area;
            int minArea = (int)GetNudValueSafe(nudMinArea);
            if (minArea < 0) minArea = 0;

            for (int i = 1; i < numLabels; i++)
            {
                int area = stats.At<int>(i, statArea);

                if (area < minArea)
                    continue;

                using Mat mask = new Mat();
                Cv2.Compare(labels, i, mask, CmpType.EQ);
                cleaned.SetTo(255, mask);
            }
        }
        finally
        {
            labels.Dispose();
            stats.Dispose();
            centroids.Dispose();
        }

        // 8) Uloženie výsledkov do zdieľaných fieldov (thread-safe).
        lock (_roiLock)
        {
            _lastContrast?.Dispose();
            _lastContrast = contrast.Clone();

            _lastBinary?.Dispose();
            _lastBinary = binary.Clone();

            _lastCleaned?.Dispose();
            _lastCleaned = cleaned.Clone();
        }

        // 9) ChangeMatrix - Resize cleaned na NxN (ponecháva sa bez zmien)
        int matrixSize = GetComboValueSafe(cboChangeMatrix);
        if (matrixSize > 0)
        {
            Mat matrix = new Mat();
            Cv2.Resize(cleaned, matrix, new OpenCvSharp.Size(matrixSize, matrixSize), 0, 0, InterpolationFlags.Area);

            lock (_roiLock)
            {
                _lastMatrix?.Dispose();
                _lastMatrix = matrix;
                matrix = null;
            }
        }
    }
    finally
    {
        // Uvoľni lokálne Mat-y (aby nevznikali memory leaky).
        upCopy?.Dispose();
        gray?.Dispose();
        smooth?.Dispose();
        contrast?.Dispose();
        // smoothGaussian?.Dispose(); // ODSTRÁNENÉ
        binary?.Dispose();
        cleaned?.Dispose();
    }
}
